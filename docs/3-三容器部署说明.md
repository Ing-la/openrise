# 三容器部署说明 (Docker Compose)

## 架构概览

| 容器 | 镜像/构建 | 职责 |
|------|-----------|------|
| **nginx** | nginx:stable-alpine | 反向代理、流量分发、静态资源缓存 |
| **frontend** | 从 ./frontend 构建 | Next.js 应用（含 API Routes） |
| **db** | postgres:15-alpine | PostgreSQL 数据库 |

## 部署步骤

### 1. 环境准备

- 确保已安装 Docker 和 Docker Compose
- 复制环境变量：
  ```bash
  cp .env.example .env
  cp frontend/.env.example frontend/.env
  ```
- 生产环境请修改：
  - `.env`：`POSTGRES_PASSWORD` 为强密码
  - `frontend/.env`：`NEXTAUTH_SECRET`、`NEXTAUTH_URL`（如 `https://yourdomain.com`）、`RESEND_*` 等

### 2. 启动服务

```bash
docker-compose up -d --build
```

### 3. 验证

- 访问 http://localhost 查看网站
- 访问 http://localhost/api/health 检查健康状态
- 访问 http://localhost/courses 查看案例库

### 4. 常用命令

```bash
# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重新构建并启动
docker-compose up -d --build
```

## 目录结构

```
ai-web-community/
├── docker-compose.yml    # 编排配置
├── .env.example         # 环境变量模板（数据库）
├── frontend/
│   ├── Dockerfile
│   ├── .env.example     # 环境变量模板（应用）
│   └── app/api/          # API 路由
└── nginx/
    └── nginx.conf        # 反向代理配置
```

## 持久化

| 数据 | Volume | 说明 |
|------|--------|------|
| 数据库 | `db_data` | PostgreSQL 数据 |
| 用户上传 | `uploads_data` | 头像、课程封面（`public/uploads/`） |

## 后续扩展

- **HTTPS**：在 `nginx/` 下添加 SSL 证书，修改 `nginx.conf` 启用 443 端口
- **Redis**：如需缓存或会话，可在 `docker-compose.yml` 中新增 service
